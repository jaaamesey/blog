> Disclaimer: This is a blog post about web*apps*, not web*sites*.
> If you're making a normal web*site* - a blog, recipe page, etc - **probably don't do anything I've suggested here**.
> Zoom gestures are a legitimate accessibility feature that should probably stick around in the places users expect them. You should only try to disable them when they're actively causing trouble, e.g. when your webapp has its own set of gestures that Safari's built-in gestures turn into an unpleasant experience.
